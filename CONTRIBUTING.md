# Contributing to DropSpot

## Branching

- Base branch: `develop` (PRs are opened to develop, merged to main after deployment).
- Feature branch: `feature/<scope>-<brief>`

## Commit messages

- Use Conventional Commits.

## PR processes

- For each PR: write a problem description, solution summary, and test steps.
- Assign at least 1 reviewer.
- Small, reviewable commits — no “big bang” commits.

## Code quality

- Lint and format: `eslint` + `prettier`
- Test requirements: backend: at least 1 unit + 1 integration; frontend: at least 2 component/e2e

## Local work

- Backend: `cd backend && npm install && npm run dev`
- Frontend: `cd frontend && npm install && npm run dev`
